给人格型 AI 搭一张
不会轻易失忆的桌子

Hippocove 想做的，不是把聊天压成几条摘要。它更像一间还在施工、但已经能开工的工坊：原始对话先落进缓存，切片、去冗余、保留溯源，再慢慢长成能放进 Obsidian 的记忆卡。

先读 README 看项目状态 技术交底 MCP / agent handoff 手测地图

它在试什么

这套仓真正难的，不是某一句 prompt，而是工作台本身：怎么让原料不丢、上下文不爆、人格种子能继续活、记忆生长能记账、原文溯源还能回去。

现在已经通到哪

  • ingest / reviewed / growth / export 主链连通
  • 共享人格工位与语言指纹可复用
  • MCP 工具可驱动任务推进
  • 旧实验台保留明文核对与调参能力

怎么看这个仓

把它当成一套可继续长的半成品工位来看，而不是最终答案。正式前台像入口，旧实验台像工坊，后端和 MCP 像心脏与机械臂。

怎么先手测

先看首页，再进旧实验台核人格工位和批次结果，最后看主卡生长和导出。别一上来只盯 UI，也别一头扎进代码。

谁适合接手

想自己调模型、调提示词、调人格种子、调旧实验台流程的人，会比只想开箱即用的人更适合它。
